    **Create `src/ct/tools/plant_data.py`:**

    New tool module with two registered tools following the exact `@registry.register` pattern from CLAUDE.md.

    **Tool 1: `data.list_datasets`**
    ```python
    @registry.register(
        name="data.list_datasets",
        description="List available local plant datasets with their species coverage and file schemas",
        category="data",
        parameters={
            "data_root": "Optional path to data root (defaults to configured data.base)"
        },
        usage_guide=(
            "Call before any data analysis to discover what curated datasets are available locally. "
            "Returns dataset names, species covered, and file schemas from manifest files."
        ),
    )
    def list_datasets(data_root: str = "", **kwargs) -> dict:
    ```

    Implementation:
    1. Lazy import: `from ct.agent.config import Config` and `from ct.data.manifest import load_manifest, manifest_summary`
    2. Resolve data root: use `data_root` param if provided, else `Config.load().get("data.base")`, else `~/.ct/data`
    3. If data root doesn't exist, return `{"summary": "No data directory found at {path}. Run 'ag data pull <dataset>' to download datasets.", "datasets": []}`
    4. Iterate subdirectories of data root
    5. For each subdirectory, call `load_manifest(subdir)`
    6. If manifest exists, include in results with `manifest_summary()`
    7. If no manifest, still list the directory with a note "No manifest — explore files directly"
    8. Return `{"summary": "Found {N} datasets in {path}\n\n{summaries}", "datasets": [list of manifest dicts or dir names]}`

    **Tool 2: `data.load_expression`**
    ```python
    @registry.register(
        name="data.load_expression",
        description="Load gene expression data from a local plant expression dataset",
        category="data",
        parameters={
            "gene": "Gene identifier (e.g., 'AT1G65480', 'Os01g0100100')",
            "species": "Species name or taxon ID (default: arabidopsis_thaliana)",
            "dataset": "Dataset name or path (default: plantexp)",
            "tissue": "Optional tissue filter (e.g., 'leaf', 'root', 'seed')",
        },
        usage_guide=(
            "Load tissue-level expression values for a gene from a local curated expression dataset. "
            "Reads manifest first to validate species coverage, then loads the expression parquet."
        ),
    )
    @validate_species(dataset_dir_kwarg="dataset_dir")
    def load_expression(gene: str, species: str = "arabidopsis_thaliana",
                        dataset: str = "plantexp", tissue: str = "", **kwargs) -> dict:
    ```

    Implementation:
    1. Lazy imports: `import pandas as pd`, `from ct.agent.config import Config`
    2. Resolve dataset path: if `dataset` is an absolute path, use it. Otherwise resolve as `{data_root}/{dataset}/`
    3. Set `kwargs["dataset_dir"] = str(dataset_path)` so the @validate_species decorator can find it
    4. Look for expression_matrix.parquet (or .csv) in the dataset directory
    5. If not found: return `{"error": f"Expression data not found at {dataset_path}. Run: ag data pull {dataset}", "summary": "..."}`
    6. Load with `pd.read_parquet(path)` (or `pd.read_csv` for CSV)
    7. Filter by gene: `df[df['gene_id'] == gene]` (case-insensitive matching on gene_id column)
    8. If tissue param provided, filter by tissue column
    9. If no rows match: return `{"summary": f"No expression data found for {gene}", "gene": gene, "n_samples": 0, "expression": []}`
    10. Build summary with tissue breakdown: group by tissue, compute mean TPM
    11. Return `{"summary": "Expression data for {gene}: {N} samples across {M} tissues\n{tissue_table}", "gene": gene, "species": species, "n_samples": len(filtered), "expression": tissue_means_as_list_of_dicts}`

    CRITICAL: Both tools must accept `**kwargs` and return a dict with `"summary"` key. Use lazy imports inside function bodies.

    2. Add `"plant_data"` to `_TOOL_MODULES` tuple. Without this, the tools won't auto-register on import.
