    **Create `src/ct/tools/_validation.py`:**

    Module providing the `@validate_species` decorator for organism validation. Follows the decorator pattern from 02-RESEARCH.md Pattern 4.

    ```python
    def validate_species(dataset_dir_kwarg: str = "dataset_dir"):
        """Decorator: warn if requested species is not covered by the dataset manifest.

        The decorator looks for `species` and the named dataset_dir kwarg in the
        function's kwargs. If the manifest for the dataset directory lists
        species_covered and the requested species is not in that list, a warning
        is included in the returned dict.

        Never blocks execution. Always proceeds with data loading.
        """
    ```

    Implementation details:

    1. Extract `species` from kwargs (if absent or empty, skip validation entirely)
    2. Extract `dataset_dir` (using the configurable kwarg name) from kwargs
    3. If both present: load manifest via `load_manifest(Path(dataset_dir))`
    4. If manifest exists and has `species_covered`:
       - Resolve the requested species to canonical binomial via `resolve_species_binomial()`
       - Compare (case-insensitive) against manifest's `species_covered` list
       - If canonical binomial is NOT in the covered list AND the canonical binomial is not the default (meaning the species was genuinely unresolved): generate a warning string
       - If the species resolved to default because it was unknown: generate a different note: "Species '{original}' is not in the registry; metadata could not be verified. Data returned anyway."
    5. Call the wrapped function normally: `result = func(*args, **kwargs)`
    6. If a warning was generated and result is a dict:
       - Add `species_warning` key to result dict
       - Prepend warning to `summary` key so agent sees it first
    7. Return result

    CRITICAL behavioral rules (from CONTEXT.md locked decisions):
    - NEVER raise an exception
    - NEVER return an error dict instead of real data
    - NEVER block execution for any reason
    - Always call the wrapped function and return its result
    - Warning is purely additive metadata on the result dict

    Use `logging.getLogger("ct.tools.validation")` for log-level warnings.
    Use `functools.wraps` to preserve function metadata.
