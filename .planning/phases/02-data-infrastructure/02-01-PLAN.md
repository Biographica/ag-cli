    **Create `src/ct/data/manifest.py`:**

    New module for loading and summarizing dataset manifests. Functions:

    1. `load_manifest(dataset_dir: Path) -> dict | None` — loads `manifest.yaml` from directory. Try `.yaml` first, then `.json` fallback. Returns None if neither exists. Uses `yaml.safe_load()`. Never raises on missing file.

    2. `manifest_species(manifest: dict) -> list[str]` — extracts `species_covered` list from manifest dict. Returns `[]` if key absent.

    3. `manifest_summary(manifest: dict) -> str` — returns human-readable multi-line string: dataset name, description, species covered, and file names. Format:
    ```
    {dataset}: {description}
    Species: {comma-separated species}
    Files: {comma-separated filenames}
    ```

    Follow existing codebase patterns: module docstring, type hints, no external dependencies beyond PyYAML and pathlib.

    **Add `ag species list` CLI command to `src/ct/cli.py`:**

    After the `data_app` section (around line 292), add a new section:
    ```
    # --- Species subcommand ---
    species_app = typer.Typer(help="Species registry management")
    ```

    Add `species list` command that:

    4. Iterates `list_all_species()` and adds rows
    5. Prints table via `console.print(table)`

    Follow the exact pattern of `data_app` and `tool_app` — lazy import inside the command function body, use existing `console` instance.
