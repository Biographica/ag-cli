**1. Create `src/ct/tools/_api_cache.py`** — shared disk TTL cache helper.

Implement three functions:
- `_cache_path(namespace: str, key: str) -> Path` — derives a stable cache file path from namespace + key. Use `hashlib.sha256(key.encode()).hexdigest()[:16]` for the filename. Cache base: `Path.home() / ".ct" / "cache"`.
- `get_cached(namespace: str, key: str, ttl_seconds: int = 86400) -> dict | None` — returns cached value or None if absent/expired. Reads JSON from `_cache_path()`, checks `_cached_at` timestamp against TTL. Returns `None` on any exception (file missing, corrupt JSON, expired). Never raises.
- `set_cached(namespace: str, key: str, value: dict) -> None` — writes value to disk cache with current timestamp. Creates parent directories (`mkdir(parents=True, exist_ok=True)`). Wraps in try/except — cache write failures are non-fatal (pass silently).

Cache file format: `{"_cached_at": <float timestamp>, "value": <dict>}`. Use `json.dumps` / `json.loads`. Only stdlib imports: `json`, `hashlib`, `time`, `pathlib.Path`.

**2. Create `src/ct/tools/interactions.py`** — STRING plant PPI tool.

Register tool:
```python
@registry.register(
    name="interactions.string_plant_ppi",
    description="Query STRING for protein-protein interactions for a plant gene. Returns interaction partners with confidence scores.",
    category="interactions",
    parameters={
        "gene": "Gene symbol or locus code (e.g. 'FLC', 'AT5G10140', 'OsMADS51')",
        "species": "Species name or taxon ID (default: Arabidopsis thaliana)",
        "min_score": "Minimum interaction confidence 0-1 (default 0.4)",
        "limit": "Max interaction partners to return (default 20, max 50)",
    },
    usage_guide="Retrieve protein interaction partners and confidence scores for a plant gene from STRING. Use for target validation, pathway context, and identifying functional partners. If a gene symbol returns no results, try the locus code instead (e.g. AT5G10140 for FLC).",
)
```

Implementation flow:
1. Lazy import `resolve_species_taxon`, `resolve_species_binomial` from `ct.tools._species`, `request_json` from `ct.tools.http_client`, `get_cached` / `set_cached` from `ct.tools._api_cache`.
2. Validate species: `taxon_id = resolve_species_taxon(species)`. If `taxon_id == 0`, return error dict with `"summary": f"Species not recognised: {species!r}. Run 'ag species list' for supported species."`.
3. Cap limit: `limit = min(int(limit), 50)`.
4. Check disk cache: `cache_key = f"string_ppi:{taxon_id}:{gene}:{min_score}:{limit}"`. Call `get_cached("string_ppi", cache_key)`. If not None, return cached value.
5. **Step 1 — Resolve gene to STRING ID:** Call `request_json("GET", "https://string-db.org/api/json/get_string_ids", params={"identifiers": gene, "species": taxon_id, "limit": 1, "caller_identity": "ag-cli"}, timeout=15, retries=2)`. If error or empty list, return structured empty response: `{"summary": f"No STRING identifier found for '{gene}' in species {binomial} (taxon {taxon_id}). Try using the locus code.", "gene": gene, "species": binomial, "taxon_id": taxon_id, "interactions": [], "query_used": gene}`.
6. Extract `string_id = ids_data[0]["stringId"]` and `preferred_name = ids_data[0].get("preferredName", gene)`.
7. **Step 2 — Fetch interaction partners:** Call `request_json("GET", "https://string-db.org/api/json/interaction_partners", params={"identifiers": string_id, "species": taxon_id, "required_score": int(min_score * 1000), "limit": limit, "caller_identity": "ag-cli"}, timeout=15, retries=2)`. If error, return error dict with summary.
8. **Build result:** Parse partner list. For each partner entry, extract: `preferredName_A`, `preferredName_B`, `score` (divide by 1000 to get 0-1 float), `stringId_A`, `stringId_B`. Determine which partner is the "other" one (the one that is not the query gene's STRING ID). Build list of partner dicts: `{"partner": other_preferred_name, "string_id": other_string_id, "score": score_float}`. Sort by score descending.
9. Construct result dict:
   ```python
   result = {
       "summary": f"Found {len(partners)} interaction partners for {preferred_name} ({gene}) in {binomial} (STRING, min_score={min_score}).",
       "gene": gene,
       "resolved_name": preferred_name,
       "string_id": string_id,
       "species": binomial,
       "taxon_id": taxon_id,
       "min_score": min_score,
       "interaction_count": len(partners),
       "interactions": partners,  # list of {partner, string_id, score}
   }
   ```
   If no interactions found, adjust summary: `f"No interaction partners found for {preferred_name} ({gene}) in {binomial} with score >= {min_score}."`.
10. Cache result: `set_cached("string_ppi", cache_key, result)`. Return result.

Follow codebase conventions: module docstring at top, `from ct.tools import registry` at module level (not lazy), everything else lazy-imported inside the function body. Accept `**kwargs`. Return dict with `"summary"` key.
