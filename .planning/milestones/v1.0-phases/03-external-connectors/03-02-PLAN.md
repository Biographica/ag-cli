**1. Add `literature.pubmed_plant_search` to `src/ct/tools/literature.py`**

Add a module-level flag near the top of the file (after imports, before first function):
```python
_pubmed_rate_limit_warned: bool = False
```

Register the tool (add after the existing `pubmed_search` function, or at the end of the file before `patent_search`):
```python
@registry.register(
    name="literature.pubmed_plant_search",
    description="Search PubMed with plant-specific query construction (species + gene). Returns structured citations with abstracts.",
    category="literature",
    parameters={
        "gene": "Gene name or symbol to search for",
        "species": "Plant species (e.g. 'Arabidopsis thaliana', 'rice'). Used to construct organism-scoped query.",
        "extra_terms": "Additional search terms to AND into the query (optional)",
        "max_results": "Maximum number of results (default 10, max 50)",
        "fetch_abstracts": "Whether to fetch full abstracts via EFetch (default True)",
    },
    usage_guide="Search PubMed for plant gene literature with automatic species-aware query construction. Returns the query used so you can refine if results are too broad or too narrow.",
)
```

Implementation flow:
1. Lazy imports: `resolve_species_binomial` from `ct.tools._species`, `get_cached`/`set_cached` from `ct.tools._api_cache`, `request_json` from `ct.tools.http_client`.
2. Cap max_results: `max_results = min(int(max_results), 50)`.
3. **Once-per-session rate limit warning:** Use `global _pubmed_rate_limit_warned`. Check `session = kwargs.get("_session")`. Get `ncbi_key = session.config.get("api.ncbi_key") if session else None`. If no key and not yet warned, set `_pubmed_rate_limit_warned = True` and store a `rate_note = "Note: NCBI API key not configured — rate limited to 3 req/s. Set via: ag config set api.ncbi_key <key>"`. Otherwise `rate_note = ""`.
4. **Construct query:** Get `binomial = resolve_species_binomial(species)`. If binomial is non-empty: `query = f'({gene}[Title/Abstract]) AND ("{binomial}"[Organism])'`. Else: `query = f'({gene}[Title/Abstract]) AND (plant[MeSH Terms])'`. If `extra_terms`: `query = f"({query}) AND ({extra_terms})"`. Normalize: `query = _normalize_pubmed_query(query)`.
5. **Check disk cache:** `cache_key = f"pubmed:{binomial or 'plant'}:{gene}:{extra_terms}:{max_results}:{fetch_abstracts}"`. Check `get_cached("pubmed", cache_key)`.
6. **ESearch:** Call `request_json("GET", "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi", params={"db": "pubmed", "term": query, "retmax": max_results, "retmode": "json", "sort": "relevance", "tool": "ag-cli", "email": "research@biographica.com", **({"api_key": ncbi_key} if ncbi_key else {})}, timeout=15, retries=2)`. If error, return `{"error": error, "summary": f"PubMed search failed: {error}", "query_used": query}`.
7. Extract PMIDs: `pmids = esearch_data.get("esearchresult", {}).get("idlist", [])`. Total count: `total_count = int(esearch_data.get("esearchresult", {}).get("count", 0))`.
8. If no PMIDs: return `{"summary": f"No PubMed results for query: {query}." + (f" {rate_note}" if rate_note else ""), "query_used": query, "total_count": 0, "articles": [], "species": binomial, "gene": gene}`.
9. **ESummary for citation metadata:** Call `request_json("GET", "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esummary.fcgi", params={"db": "pubmed", "id": ",".join(pmids), "retmode": "json", "tool": "ag-cli", "email": "research@biographica.com", **({"api_key": ncbi_key} if ncbi_key else {})}, timeout=15, retries=2)`. Parse each PMID's metadata from `summary_data["result"]` dict (keys are PMID strings). Extract: `title`, `sortfirstauthor` (or `authors[0]["name"]`), `fulljournalname` (or `source`), `pubdate`, `elocationid` (DOI).
10. **EFetch for abstracts (if `fetch_abstracts`):** Call `request("GET", "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi", params={"db": "pubmed", "id": ",".join(pmids), "retmode": "xml", "rettype": "abstract", "tool": "ag-cli", "email": "research@biographica.com", **({"api_key": ncbi_key} if ncbi_key else {})}, timeout=30, retries=2)`. Parse XML response with `xml.etree.ElementTree` (import lazily). For each `PubmedArticle`, find `.//AbstractText` elements and join their text with `" "`. Build a `pmid_to_abstract` dict mapping PMID string to abstract text. If EFetch fails, set all abstracts to `""` — don't fail the whole tool.
11. **Build articles list:** For each PMID, build: `{"pmid": pmid, "title": title, "abstract": pmid_to_abstract.get(pmid, ""), "authors": first_author, "journal": journal, "year": year_from_pubdate, "doi": doi}`.
12. **Build result:**
    ```python
    result = {
        "summary": f"Found {total_count} PubMed results for '{gene}' in {binomial or 'plants'} (showing {len(articles)})" + (f". {rate_note}" if rate_note else "."),
        "query_used": query,
        "total_count": total_count,
        "articles": articles,
        "species": binomial,
        "gene": gene,
    }
    ```
13. Cache and return: `set_cached("pubmed", cache_key, result)`. Return result.

**2. Add `literature.lens_patent_search` to `src/ct/tools/literature.py`**

Register:
```python
@registry.register(
    name="literature.lens_patent_search",
    description="Search Lens.org for patents related to a plant gene or trait. Supports gene-focused (narrow) and landscape (broad) modes.",
    category="literature",
    parameters={
        "query_text": "Primary search term — gene name for gene mode, or crop/trait for landscape mode",
        "mode": "Query mode: 'gene' (gene AND species, default) or 'landscape' (crop AND trait)",
        "species": "Species for gene mode (default: Arabidopsis thaliana). Ignored in landscape mode.",
        "trait": "Trait term for landscape mode (e.g. 'drought tolerance'). Required for landscape mode.",
        "max_results": "Maximum number of results (default 10, max 20)",
    },
    usage_guide="Search Lens.org for plant-related patents. Use 'gene' mode for specific gene/target patent assessment. Use 'landscape' mode with crop + trait for freedom-to-operate analysis. Requires api.lens_key to be configured.",
)
```

Implementation flow:
1. Lazy imports: `get_cached`/`set_cached` from `ct.tools._api_cache`, `request` from `ct.tools.http_client`, `resolve_species_binomial` from `ct.tools._species`.
2. Get Lens key: `session = kwargs.get("_session")`. `lens_key = session.config.get("api.lens_key") if session else None`. If no key (fallback safety — tool should be hidden but guard anyway): return `{"error": "Lens.org API key not configured. Run: ag config set api.lens_key <key>", "summary": "Lens.org API key not configured."}`.
3. Cap max_results: `max_results = min(int(max_results), 20)`.
4. **Build query string based on mode:**
   - `gene` mode (default): `binomial = resolve_species_binomial(species)`. `query_string = f'("{query_text}") AND ("{binomial}")'`. If binomial is empty, use just `f'("{query_text}") AND ("plant")'`.
   - `landscape` mode: Require `trait` param. `query_string = f'("{query_text}") AND ("{trait}")'`. If `trait` is empty/missing, return error: `{"error": "Landscape mode requires 'trait' parameter.", "summary": "Missing 'trait' parameter for landscape mode."}`.
5. **Check disk cache:** `cache_key = f"lens:{mode}:{query_string}:{max_results}"`. Check `get_cached("lens_patents", cache_key)`.
6. **API call:** Build payload:
   ```python
   payload = {
       "query": {"query_string": {"query": query_string, "fields": ["title", "abstract", "claim"]}},
       "include": ["lens_id", "title", "abstract", "claim", "applicant", "publication_date", "doc_number", "jurisdiction", "kind"],
       "size": max_results,
       "sort": [{"relevance": "desc"}],
   }
   ```
   Call `request("POST", "https://api.lens.org/patent/search", json=payload, headers={"Authorization": f"Bearer {lens_key}", "Content-Type": "application/json"}, timeout=30, retries=2, raise_for_status=False)`. Check response status: if status >= 400, return error dict with status code and body snippet.
7. **Parse response:** `resp_data = resp.json()`. Extract `total = resp_data.get("total", 0)` and `results = resp_data.get("data", [])`. For each patent result, extract:
   - `lens_id`
   - `title` (may be list — take first entry's `text` if list, else string)
   - `abstract` (similar handling — take first entry's `text` if list, join if multiple)
   - `claims`: take first 3 claim texts. Claims are `claim` field — handle as list of objects with `"text"` key, or list of strings. Use defensive access: `[c.get("text", str(c)) if isinstance(c, dict) else str(c) for c in claims_raw[:3]]`.
   - `applicants`: list of applicant names
   - `publication_date`, `doc_number`, `jurisdiction`
8. Build patents list and result:
   ```python
   result = {
       "summary": f"Found {total} patents matching '{query_string}' on Lens.org (showing {len(patents)}).",
       "query_used": query_string,
       "mode": mode,
       "total_count": total,
       "patents": patents,  # list of patent dicts
   }
   ```
   If no results: summary = `f"No patents found for query: {query_string} on Lens.org."`.
9. Cache and return: `set_cached("lens_patents", cache_key, result)`.
