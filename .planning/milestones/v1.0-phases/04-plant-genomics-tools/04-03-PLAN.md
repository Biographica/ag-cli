Implementation flow for `gff_parse(gene, species="Arabidopsis thaliana", gff_path=None, transcript=None, force=False, **kwargs) -> dict`:

1. **Lazy imports:** `resolve_species_taxon`, `resolve_species_binomial` from `ct.tools._species`; `request` from `ct.tools.http_client`. Also: `import gffutils` (lazy, inside function body), `from pathlib import Path`, `import gzip` (all lazy inside function body). Import `_CACHE_BASE` from `ct.tools._api_cache`.

2. **Species validation** — same pattern as other tools.

3. **File acquisition:**
   - If `gff_path` is provided: use `Path(gff_path)` directly. If file does not exist, return error dict.
   - If `gff_path` is not provided: construct Ensembl Plants FTP URL.
     - `species_url = binomial.lower().replace(" ", "_")` (e.g. `arabidopsis_thaliana`)
     - `species_cap = binomial.replace(" ", "_")` (e.g. `Arabidopsis_thaliana`) — use `resolve_species_binomial` result directly, just replace space with underscore. The binomial is returned with exact stored casing from the registry.
     - Get `genome_build` from registry: call `from ct.tools._species import _build_lookup; lookup = _build_lookup(); entry = lookup.get(species.lower(), (0, "", ""))`. Use `genome_build = entry[2]` if available. If empty, return error requesting user to provide `gff_path`.
     - Ensembl release: use 62 as default. Construct URL: `f"https://ftp.ensemblgenomes.ebi.ac.uk/pub/plants/release-62/gff3/{species_url}/{species_cap}.{genome_build}.62.gff3.gz"`
     - Cache path: `gff_cache_dir = _CACHE_BASE / "gff3"`. Check if `gff_cache_dir / f"{species_url}.gff3"` exists. If not, download:
       - Call `request("GET", url, timeout=120, retries=1)`. If error or status >= 400, return structured error with the URL and suggestion to download manually.
       - Decompress gzip: `gzip.decompress(resp.content)`. Write to `gff_cache_dir / f"{species_url}.gff3"`.
       - Set download timeout to 120 seconds. For oversized species (wheat >1GB), return error: `"GFF3 for {binomial} is very large. Manual download recommended: {url}. Place at {cache_path}."`.
     - Set `gff_local = gff_cache_dir / f"{species_url}.gff3"`.

4. **gffutils database creation (or load cached):**
   - `db_path = gff_local.with_suffix(".db")` (cache .db alongside .gff3)
   - If `db_path.exists()`: `db = gffutils.FeatureDB(str(db_path))`
   - Else: `db = gffutils.create_db(str(gff_local), dbfn=str(db_path), force=True, merge_strategy="merge", disable_infer_genes=True, disable_infer_transcripts=True)`
   - Wrap in try/except: if gffutils raises any error, return structured error dict.

5. **Gene lookup (ID then Name fallback):**
   - Try `gene_feature = db[gene]`
   - On `gffutils.FeatureNotFoundError`: try lookup with `gene:` prefix (Ensembl GFF3 uses `ID=gene:AT5G10140`): `gene_feature = db[f"gene:{gene}"]`
   - On second failure: fallback to Name attribute search: iterate `db.features_of_type("gene")` and find entries where `gene` appears in `feature.attributes.get("Name", [])`. Take first match.
   - If still not found: return `{"summary": f"Gene '{gene}' not found in GFF3 file.", "gene": gene, "error": "Gene not found in GFF3"}`.

6. **Extract transcript features:**
   - Get mRNAs: `mrnas = sorted(db.children(gene_feature, featuretype="mRNA"), key=lambda f: f.start)`
   - If `transcript` param given: find matching mRNA by ID. Else: use `mrnas[0]` (primary transcript).
   - If no mRNAs: try `featuretype="transcript"` as fallback (some GFF3 files use "transcript" not "mRNA").
   - If still no transcript features: return error dict.

7. **Extract exons, UTRs, compute introns:**
   ```python
   exons = sorted(db.children(selected_mrna, featuretype="exon"), key=lambda f: f.start)
   five_utrs = sorted(db.children(selected_mrna, featuretype="five_prime_UTR"), key=lambda f: f.start)
   three_utrs = sorted(db.children(selected_mrna, featuretype="three_prime_UTR"), key=lambda f: f.start)

   exon_list = [{"start": e.start, "end": e.end, "length": e.end - e.start + 1} for e in exons]
   five_utr_list = [{"start": u.start, "end": u.end, "length": u.end - u.start + 1} for u in five_utrs]
   three_utr_list = [{"start": u.start, "end": u.end, "length": u.end - u.start + 1} for u in three_utrs]

   # Compute introns from exon gaps
   introns = []
   for i in range(len(exons) - 1):
       intron_start = exons[i].end + 1
       intron_end = exons[i + 1].start - 1
       if intron_end >= intron_start:
           introns.append({"start": intron_start, "end": intron_end, "length": intron_end - intron_start + 1})
   ```

8. **Build result:**
   ```python
   result = {
       "summary": f"Gene structure for {gene} ({gene_feature.id}): {len(exon_list)} exon(s), {len(introns)} intron(s), {gene_feature.end - gene_feature.start + 1:,} bp span.",
       "gene": gene,
       "gene_id": gene_feature.id,
       "transcript": selected_mrna.id,
       "species": binomial,
       "chromosome": gene_feature.seqid,
       "strand": gene_feature.strand,
       "gene_start": gene_feature.start,
       "gene_end": gene_feature.end,
       "gene_span_bp": gene_feature.end - gene_feature.start + 1,
       "total_exons": len(exon_list),
       "total_introns": len(introns),
       "exons": exon_list,
       "introns": introns,
       "five_prime_utrs": five_utr_list,
       "three_prime_utrs": three_utr_list,
   }
   ```
   Return result (no disk caching needed — the .db file IS the cache).

```python
        "Find genes that are co-expressed with a query gene using ATTED-II data. "

    ),

```

Implementation flow for `coexpression_network(gene, species="Arabidopsis thaliana", top_n=20, mr_threshold=30.0, force=False, **kwargs) -> dict`:

1. **Lazy imports:** `resolve_species_taxon`, `resolve_species_binomial` from `ct.tools._species`; `request` from `ct.tools.http_client`; `get_cached`, `set_cached`, `_CACHE_BASE` from `ct.tools._api_cache`.

2. **Species validation** — same pattern.

3. **Cap top_n:** `top_n = min(int(top_n), 100)`.

4. **Cache check:** `cache_key = f"coexp:{taxon_id}:{gene}:{top_n}:{mr_threshold}"`. Call `get_cached("atted_coexp", cache_key, ttl_seconds=604800)` (7-day TTL for bulk data).

5. **Load ATTED-II data:**
   - `atted_dir = _CACHE_BASE / "atted"`. Create if needed: `atted_dir.mkdir(parents=True, exist_ok=True)`.
   - `species_key = binomial.lower().replace(" ", "_")` (e.g. `arabidopsis_thaliana`).
   - `atted_file = atted_dir / f"{species_key}_coexp.tsv"`.
   - **If file does not exist, attempt download:**
     - ATTED-II download URL is unstable (documented risk in research). Use a configurable base URL:
       ```python
       ATTED_DOWNLOAD_URLS = {
           "arabidopsis_thaliana": "https://atted.jp/download/Ath-r.c7-0.MR.lst.gz",
           "oryza_sativa": "https://atted.jp/download/Osa-r.c7-0.MR.lst.gz",
       }
       ```
     - If species_key in `ATTED_DOWNLOAD_URLS`: attempt download using `request("GET", url, timeout=120, retries=1)`.
     - On success: decompress gzip, write to `atted_file`.
     - On failure (any error): return a structured fallback result:
       ```python
       return {
           "summary": f"ATTED-II co-expression data unavailable for {binomial}. Download may have failed — ATTED-II URLs change between versions. Try downloading manually from https://atted.jp and placing at {atted_file}.",
           "gene": gene,
           "species": binomial,
           "coexpressed_genes": [],
           "cluster_size": 0,
           "data_source": "ATTED-II (unavailable)",
           "fallback": True,
       }
       ```
     - If `species_key` not in `ATTED_DOWNLOAD_URLS`: return similar fallback with message that only Arabidopsis (and rice best-effort) are supported.

6. **Parse ATTED-II file and query gene:**
   - Read the file. ATTED-II format: tab-delimited, each line has `gene_a\tgene_b\tMR_score` (or similar column layout — parse defensively).
   - Import `pandas` lazily: `import pandas as pd`.
   - `df = pd.read_csv(atted_file, sep="\t", header=None, names=["gene_a", "gene_b", "mr_score"], comment="#", usecols=[0, 1, 2])` — read only first 3 columns, skip comment lines. Wrap in try/except; if parsing fails, try alternative column layouts (the exact format may vary — handle gracefully).
   - Filter to rows where `gene_a == gene` or `gene_b == gene` (case-insensitive match on locus codes):
     ```python
     gene_upper = gene.upper()
     mask = (df["gene_a"].str.upper() == gene_upper) | (df["gene_b"].str.upper() == gene_upper)
     gene_df = df[mask].copy()
     ```
   - For each row, determine the "partner" (the gene that is NOT the query gene). Compute: `{"partner": partner, "mr_score": mr_score}`.
   - Sort by MR score ascending (lower = stronger co-expression).
   - Take top `top_n` entries.

7. **Compute cluster membership:**
   - Count entries with `mr_score <= mr_threshold` — this is the cluster size.

8. **Build result:**
   ```python
   result = {
       "summary": f"Found {len(coexpressed)} co-expression partner(s) for {gene} in {binomial} (ATTED-II, top {top_n}, MR threshold {mr_threshold}).",
       "gene": gene,
       "species": binomial,
       "taxon_id": taxon_id,
       "top_n": top_n,
       "mr_threshold": mr_threshold,
       "coexpressed_genes": coexpressed,  # list of {partner, mr_score}
       "cluster_size": cluster_count,     # genes within MR threshold
       "data_source": "ATTED-II",
   }
   ```
   If no partners found: summary = `f"No co-expression data found for {gene} in ATTED-II ({binomial}). Ensure gene locus code format (e.g. AT5G10140, not gene symbol)."`.
   Cache with `set_cached("atted_coexp", cache_key, result)`. Return result.
