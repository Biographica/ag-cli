**1. Add `genomics.gene_annotation` tool to `src/ct/tools/genomics.py`**

Append the following tool AFTER the existing `gwas_lookup` function (do not modify existing code). The new tool uses lazy imports inside the function body (not module-level) for `_species`, `_api_cache`, and `http_client`.

Register:
```python
@registry.register(
    name="genomics.gene_annotation",
    description=(
        "Look up gene annotation (GO terms, functional description, linked publications) "
        "for a gene in any supported plant species using Ensembl Plants and UniProt."
    ),
    category="genomics",
    parameters={
        "gene": "Gene symbol or locus code (e.g. 'FLC', 'AT5G10140', 'GW5')",
        "species": "Species name (default: Arabidopsis thaliana)",
        "force": "Skip species registry check and try any species string (default: False)",
    },
    usage_guide=(
        "Get GO terms, functional description, and linked publications for a plant gene. "
        "Start here for target characterisation. Cross-reference PubMed IDs with "
        "literature.pubmed_plant_search for full text."
    ),
)
```

Implementation flow for `gene_annotation(gene, species="Arabidopsis thaliana", force=False, **kwargs) -> dict`:

1. **Lazy imports:** `resolve_species_taxon`, `resolve_species_binomial` from `ct.tools._species`; `request_json` from `ct.tools.http_client`; `get_cached`, `set_cached` from `ct.tools._api_cache`.

2. **Species validation (with force escape hatch):**
   - `taxon_id = resolve_species_taxon(species)`
   - If `taxon_id == 0` and `not force`: return `{"error": f"Unknown species: {species!r}. Use force=True to override.", "summary": f"Species not recognised: {species!r}."}`
   - If `taxon_id == 0` and `force`: set `taxon_id = 0` (proceed with raw species string for URLs)
   - `binomial = resolve_species_binomial(species) or species`

3. **Cache check:** `cache_key = f"gene_annotation:{taxon_id}:{gene}"`. Call `get_cached("ensembl_gene", cache_key)`. If not None, return cached value.

4. **Step 1 — Ensembl Plants gene lookup by symbol:**
   - `ensembl_base = "https://rest.ensembl.org"`
   - `species_url = binomial.lower().replace(" ", "_")` (e.g. `"arabidopsis_thaliana"`)
   - Call `request_json("GET", f"{ensembl_base}/lookup/symbol/{species_url}/{gene}", params={"content-type": "application/json"}, timeout=15, retries=2)`
   - If error or None: return `{"summary": f"Gene '{gene}' not found in Ensembl Plants for {binomial}.", "gene": gene, "species": binomial, "error": err or "Not found"}`.
   - Extract: `ensembl_id = gene_data.get("id", "")`, `description = gene_data.get("description", "")`, `display_name = gene_data.get("display_name", gene)`, `biotype = gene_data.get("biotype", "")`, `chromosome = gene_data.get("seq_region_name", "")`, `start = gene_data.get("start")`, `end = gene_data.get("end")`, `strand = gene_data.get("strand")`.

5. **Step 2 — GO cross-references from Ensembl:**
   - Call `request_json("GET", f"{ensembl_base}/xrefs/id/{ensembl_id}", params={"content-type": "application/json", "external_db": "GO", "all_levels": 0}, timeout=15, retries=2)`
   - Parse response: build list of unique GO terms. For each xref in `(go_data or [])` where `xref.get("primary_id", "").startswith("GO:")`:
     ```python
     {"go_id": xref["primary_id"], "term": xref.get("description", ""), "evidence": xref.get("info_type", "")}
     ```
   - Deduplicate by `go_id` (keep first occurrence).

6. **Step 3 — UniProt for protein-level GO + publications:**
   - Call `request_json("GET", "https://rest.uniprot.org/uniprotkb/search", params={"query": f"gene:{gene} AND organism_id:{taxon_id} AND reviewed:true", "fields": "gene_names,go,cc_function,lit_pubmed_id", "format": "json", "size": 1}, timeout=15, retries=2)`
   - If results exist (`(uniprot_data or {}).get("results", [])`):
     - Extract UniProt GO terms from `uniProtKBCrossReferences` where `database == "GO"`. Each has `id` (GO:XXXXXXX) and `properties` list with `key: "GoTerm"` containing `value: "C:..."`, `"F:..."`, or `"P:..."`. Parse namespace from prefix: C=cellular_component, F=molecular_function, P=biological_process.
     - Extract PubMed IDs from `references` list: for each ref, check `citation.citationCrossReferences` for entries with `database == "PubMed"`, extract `id`. Also extract `citation.title` if available.
     - Extract function description from `comments` list where `commentType == "FUNCTION"` — join text values.
   - Merge UniProt GO terms into the Ensembl GO list (deduplicate by go_id).

7. **Build result:**
   ```python
   result = {
       "summary": f"Gene annotation for {display_name} ({ensembl_id}) in {binomial}: {len(go_terms)} GO terms, {len(pubmed_ids)} linked publications.",
       "gene": gene,
       "ensembl_id": ensembl_id,
       "display_name": display_name,
       "species": binomial,
       "taxon_id": taxon_id,
       "description": description,
       "biotype": biotype,
       "location": {"chromosome": chromosome, "start": start, "end": end, "strand": strand},
       "go_terms": go_terms,            # list of {go_id, term, evidence, namespace}
       "function_description": uniprot_function or description,
       "pubmed_ids": pubmed_ids,         # list of {pmid, title}
       "pubmed_count": len(pubmed_ids),
   }
   ```
   Cache with `set_cached("ensembl_gene", cache_key, result)`. Return result.

**NOTE:** Use defensive `.get()` access throughout — Ensembl and UniProt responses may have missing fields. Never raise on missing data; return what is available.

**2. Add `genomics.gwas_qtl_lookup` tool to `src/ct/tools/genomics.py`**

Append after `gene_annotation`. Same lazy import pattern.

Register:
```python
@registry.register(
    name="genomics.gwas_qtl_lookup",
    description=(
        "Look up GWAS hits and QTL/phenotype annotations for a gene in a plant species "
        "using the Ensembl Plants phenotype endpoint."
    ),
    category="genomics",
    parameters={
        "gene": "Gene symbol or locus code (e.g. 'FLC', 'AT5G10140', 'GW5')",
        "species": "Species name (default: Arabidopsis thaliana)",
        "trait": "Optional trait keyword to filter results (e.g. 'flowering time', 'yield')",
        "force": "Skip species registry check (default: False)",
    },
    usage_guide=(
        "Look up GWAS hits and phenotype/QTL evidence for a plant gene. "
        "Arabidopsis has the richest coverage. For other species, try querying "
        "the Arabidopsis ortholog if results are sparse."
    ),
)
```

Implementation flow for `gwas_qtl_lookup(gene, species="Arabidopsis thaliana", trait=None, force=False, **kwargs) -> dict`:

1. **Lazy imports** — same as `gene_annotation`.

2. **Species validation** — same pattern as `gene_annotation` (taxon_id=0 + no force → error; force → proceed).

3. **Cache check:** `cache_key = f"gwas_qtl:{taxon_id}:{gene}:{trait or ''}"`. Call `get_cached("ensembl_phenotype", cache_key)`.

4. **Step 1 — Resolve gene to Ensembl ID** (needed for URL):
   - `species_url = binomial.lower().replace(" ", "_")`
   - Call `request_json("GET", f"{ensembl_base}/lookup/symbol/{species_url}/{gene}", params={"content-type": "application/json"}, timeout=15, retries=2)`
   - If error: try using `gene` directly as the identifier in the phenotype URL (some genes may be locus codes that work directly).

5. **Step 2 — Ensembl phenotype/gene endpoint:**
   - Call `request_json("GET", f"{ensembl_base}/phenotype/gene/{species_url}/{gene}", params={"content-type": "application/json", "include_associated": 1, "include_pubmed_id": 1, "non_specified": 1}, timeout=15, retries=2)`
   - If error or empty list: build empty result with suggestion.

6. **Step 3 — Parse and optionally filter by trait:**
   - For each phenotype entry in `(phenotype_data or [])`:
     ```python
     {
         "description": entry.get("description", ""),
         "source": entry.get("source", ""),
         "study": entry.get("study", ""),
         "pubmed_id": entry.get("pubmed_id", ""),
         "attributes": entry.get("attributes", {}),
     }
     ```
   - If `trait` is provided: filter to entries where `trait.lower()` appears in `entry["description"].lower()`.
   - Sort by whether pubmed_id is present (entries with PubMed evidence first).

7. **Build result:**
   ```python
   suggestion = ""
   if not phenotypes and species_url != "arabidopsis_thaliana":
       suggestion = (
           f"No phenotype annotations found for {gene} in {binomial}. "
           "Arabidopsis has the richest phenotype data in Ensembl Plants. "
           "Try looking up the Arabidopsis ortholog using genomics.ortholog_map."
       )
   elif not phenotypes:
       suggestion = (
           f"No phenotype annotations found for {gene} in Ensembl Plants. "
           "GWAS evidence for plant traits is sparser than for human diseases."
       )

   result = {
       "summary": f"Found {len(phenotypes)} phenotype annotation(s) for {gene} in {binomial}" +
                  (f" matching '{trait}'" if trait else "") + "." +
                  (f" {suggestion}" if suggestion and not phenotypes else ""),
       "gene": gene,
       "species": binomial,
       "taxon_id": taxon_id,
       "trait_filter": trait,
       "phenotype_count": len(phenotypes),
       "phenotypes": phenotypes,
       "suggestion": suggestion if not phenotypes else "",
   }
   ```
   Cache with `set_cached("ensembl_phenotype", cache_key, result)`. Return result.
