1. Create a new module `src/ct/tools/_species.py` with the interim species resolution helper. Use the pattern from 01-RESEARCH.md Pattern 2 as the foundation:
   - `_PLANT_TAXON_MAP` dict mapping binomial names to NCBI taxon IDs, plus common names and abbreviations to canonical binomial forms
   - Cover at minimum: Arabidopsis thaliana (3702), Oryza sativa (4530), Zea mays (4577), Solanum lycopersicum (4081), Solanum tuberosum (4113), Triticum aestivum (4565), Glycine max (3847), Brassica napus (3708), Nicotiana tabacum (4097), Populus trichocarpa (3694)
   - Include common name aliases: arabidopsis/at, rice/os, maize/corn/zm, tomato, potato, wheat, soybean, oilseed rape/canola, tobacco, poplar
   - Retain human (9606) and mouse (10090) entries for cross-species lookups on general tools
   - `resolve_species_taxon(species: str, default_taxon: int = 3702) -> int` — resolves any species string to NCBI taxon ID
   - `resolve_species_binomial(species: str, default: str = "Arabidopsis thaliana") -> str` — resolves any species string to canonical binomial
   - Handle edge cases: numeric taxon ID input (pass through as int), None/empty (return default), unknown species (return default)

**Update surviving-category tools with hardcoded species:**

2. **src/ct/tools/network.py** — Find all functions that hardcode `9606` or `"human"` as species. For each:
   - Add `species: str = "Arabidopsis thaliana"` parameter to function signature (before `**kwargs`)
   - Add `"species": "Species name or taxon ID (default: Arabidopsis thaliana)"` to the `parameters` dict in `@registry.register()`
   - Inside the function body, add lazy import: `from ct.tools._species import resolve_species_taxon`
   - Replace hardcoded `9606` with `resolve_species_taxon(species)`
   - Update the `usage_guide` to mention plant species if present

3. **src/ct/tools/data_api.py** — Same pattern. Find UniProt, Ensembl, MyGene lookups that default to human. Add species parameter, use `resolve_species_taxon()` or `resolve_species_binomial()` as appropriate (some APIs use taxon ID, others use binomial name in URLs).

4. **src/ct/tools/protein.py** — Same pattern. Find `organism_id: 9606` or similar hardcoded references. Replace with species parameter + resolution.

5. **src/ct/tools/genomics.py** — Same pattern. Find hardcoded `homo_sapiens` in Ensembl REST API URLs. Replace with `resolve_species_binomial(species).lower().replace(" ", "_")` for URL construction.

6. **src/ct/tools/parity.py** — Same pattern. Find `_normalize_mygene_species` defaults or similar. Update to use plant defaults.

**IMPORTANT guidelines:**
- Only modify tools in categories that survive the PLANT_SCIENCE_CATEGORIES allowlist (per Plan 01). Tools in excluded categories (chemistry, clinical, etc.) are invisible to the agent and should NOT be modified.
- Use lazy imports (`from ct.tools._species import resolve_species_taxon` inside function body) per codebase convention.
- Keep `**kwargs` in all function signatures.
- Default species is `"Arabidopsis thaliana"` (the model plant) per research recommendation — consistent with how original tools defaulted to "human".
- Run `grep -rn '9606\|homo.sapiens\|"human"\|species.*human' src/ct/tools/` to find ALL hardcoded references in surviving categories. The 5 files listed above are the known ones from research, but there may be others.
